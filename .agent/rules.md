# Правила разработки (Project Rules)

## 1. Организация кода (Code Style)
- Строгое следование стандарту **PEP 8** (использовать `black` или `autopep8` для форматирования).
- Все публичные функции и классы должны содержать **docstrings** на русском языке (в формате Google или Sphinx).
- Максимальная длина строки: **100 символов** (для улучшения читаемости на широких экранах).

## 2. Безопасность и конфигурация
- **НИКОГДА** не хардкодить токены и ключи в коде.
- Все секреты хранятся в файле `.env` (добавлен в `.gitignore`).
- Для работы с переменными окружения используется библиотека `python-dotenv`.
- Управление зависимостями ведется строго через файл `requirements.txt`.

## 3. Логирование и обработка ошибок
- Все ошибки и информационные сообщения логировать в файл `logs/app.log` с ротацией.
- При ошибках API (Epic Games) реализовать повторные попытки (retry) с экспоненциальной задержкой.
- Бот не должен падать (crash) при отсутствии сети; должен корректно обрабатывать исключения.

## 4. Структура проекта (рекомендуемая)
- `/src` — исходный код бота.
- `/src/api` — модули для работы с Epic Games API.
- `/src/bot` — модули для работы с Telegram (отправка сообщений).
- `/logs` — директория для логов.
- `/tests` — модульные тесты (pytest).

## 5. Работа с Git
- Commit'ы должны быть атомарными и начинаться с глагола (например, `feat: add parser for games`, `fix: retry logic`).
- Ветка для разработки: `develop`, продакшн: `main`.
