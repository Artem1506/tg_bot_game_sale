# 🎮 Epic Free Games Bot

Telegram-бот на базе `aiogram 3.x` для автоматического отслеживания и публикации еженедельных бесплатных игр из Epic Games Store в ваш Telegram-канал.

**Бот:** [@gamefreebie_bot](https://t.me/gamefreebie_bot)  
**Канал:** [@grampsgamer](https://t.me/grampsgamer)

---

## 📌 О проекте

Бот предназначен для мониторинга бесплатных раздач игр в **Epic Games Store** (автоматически по расписанию) и бесплатных ассетов на маркетплейсе **Fab.com** (по ручному запросу администратора), формируя красивые структурированные публикации для Telegram-канала.

### Ключевые особенности:
* **Объединение игр (Media Group):** Если в раздаче EGS участвуют несколько игр (обычно две), бот отправляет их **одним сообщением**. Картинки собираются в альбом (Media Group), а описания и ссылки публикуются как единая подпись под первой картинкой.
* **Раздачи FAB.com:** Мониторинг раздач ассетов Unreal Engine/Unity на маркетплейсе Fab. Бот парсит страницу раздач Fab.com и автоматически собирает краткие описания со страниц каждого ассета, оформляя их в аккуратный список с указанием авторов.
* **Обход блокировок и Cloudflare:** Запросы к сайту Fab.com выполняются асинхронно через встроенную системную утилиту `curl`, что позволяет стабильно получать контент и обходить защиту Cloudflare без использования тяжелых браузерных движков.
* **Защита от дубликатов:** Бот ведет локальную базу отправленных раздач в JSON-файле (`history.json`), чтобы исключить повторную публикацию одних и тех же игр.
* **Два режима работы:** 
  1. *Постоянный запуск (Демон):* Бот запущен непрерывно и сам отсчитывает время до проверки.
  2. *Разовый запуск (Планировщик):* Скрипт запускается один раз, делает проверку и завершает процесс (экономит память ПК).

## ⚙️ Доступные команды бота

Команды работают в личных сообщениях с ботом. Административные команды доступны только пользователям, чьи ID прописаны в параметре `ADMIN_IDS` в файле `.env`.

### Для пользователей:
* `/start` — Запуск бота. Выводит приветственное сообщение, статус пользователя (пользователь/администратор) и показывает личный Telegram ID.

### Для администраторов:
* `/check` — **Безопасное превью игр.** Запускает немедленный поиск раздач в Epic Games API и присылает готовый пост (альбом) **напрямую в ЛС администратору** (в канал публикации нет).
* `/check channel` — **Публикация игр в канал.** Запускает поиск раздач и принудительно публикует их в Telegram-канал, игнорируя историю `history.json`.
* `/check_fab` — **Превью раздач FAB.** Парсит страницу раздач Fab.com, собирает описания ассетов и отправляет красиво оформленный список со ссылками и авторами **напрямую в ЛС администратору**.
* `/check_fab channel` — **Публикация FAB в канал.** Запускает проверку раздач FAB и отправляет готовый список ассетов в Telegram-канал.
* `/help` — Выводит справку по административным командам.

## 🚀 Варианты запуска бота

Перед запуском установите библиотеки из файла `requirements.txt`:
```bash
py -m pip install -r requirements.txt
```

### Вариант 1. Разовый запуск раз в день через Планировщик Windows (Рекомендуется для домашнего ПК)
Этот способ не требует, чтобы бот постоянно висел в памяти компьютера. 

1. Настройте файл `.env`.
2. Настройте запуск файла `run_once.bat` через **Планировщик задач Windows** (Task Scheduler):
   * Создайте простую задачу с триггером "Ежедневно".
   * В поле "Программа или сценарий" укажите путь к файлу: `...\tg_bot_game_sale\run_once.bat`.
   * В поле "Рабочая папка" укажите путь к корню проекта: `...\tg_bot_game_sale`.
3. Каждый день в указанное время скрипт `src/run_once.py` запустится на 3 секунды, опубликует новые игры (если они появились) и автоматически закроется.

### Вариант 2. Постоянный запуск (Демон)
Подходит для хостинга на сервере (VPS) или если компьютер работает 24/7.

1. Запустите файл `src/main.py`:
   ```bash
   py src/main.py
   ```
2. Бот будет работать непрерывно, обрабатывать команды пользователей в реальном времени и раз в день в установленное в `CHECK_TIME` время автоматически отправлять раздачи.

## 🛠️ Настройка окружения (`.env`)

Создайте или отредактируйте файл `.env` в корне проекта:
```env
# Токен бота от @BotFather
BOT_TOKEN=ваш_токен_бота

# ID или юзернейм канала для публикаций
CHANNEL_ID=...

# Список ID администраторов бота через запятую (для работы команды /check)
ADMIN_IDS=...

# Время ежедневной автоматической проверки раздач (в формате HH:MM, только для Варианта 2)
CHECK_TIME=19:00
```
> ⚠️ **Важно:** Любые изменения в файле `.env` вступают в силу только после перезапуска бота/скрипта.
